Columns that appear in schema.sql CREATE statements but were NEVER added by

P0-1  SCHEMA DRIFT: users.token_version — CAN 500 EVERY AUTHENTICATED REQUEST
================================================================================
WHERE: functions/lib/auth.ts:15-21 (requireAuth)

  const user = await db.prepare("SELECT token_version FROM users WHERE id = ?")
    .bind(payload.userId).first();
  if (!user || (payload.tokenVersion ?? 0) !== (user.token_version ?? 0)) {
    return json({ error: "Session expired" }, 401);
  }

requireAuth runs this D1 query on EVERY authenticated request. If the
production users table lacks token_version (no migration ever added it),
D1 throws "no such column: token_version", the catch block returns
500 "Server configuration error" — for EVERY logged-in API call: dashboard,
events, uploads, admin, everything. This single line can account for
"many things are not working" by itself.

P1-7  ENTERING A GATE KEY LOGS THE REAL USER OUT
================================================================================
WHERE: src/pages/SpacePage.tsx:93-94

  const data = await api.post<...>("/auth/gate", { spaceSlug, gateKey });
  setToken(data.token);

setToken() writes to the single localStorage key "photo_token" — the same one
that holds a logged-in user's JWT. A signed-in owner who visits a friend's
gated space and enters the key silently replaces their own session with a
role="viewer" gate token; on the next reload /auth/me returns user:null and
they're logged out.

FIX (minimal, no api.ts refactor): store gate tokens separately per space and
only use the global token slot when nobody is logged in:

  In src/lib/api.ts add:
    export function setGateToken(spaceSlug: string, token: string) {
      localStorage.setItem(`photo_gate_${spaceSlug}`, token);
    }
    export function getGateToken(spaceSlug: string): string | null {
      return localStorage.getItem(`photo_gate_${spaceSlug}`);
    }
    // optional: an override the caller can pass per-request
    export const api = { get<T>(path, opts?: { token?: string }) ... }
    (thread opts.token into the Authorization header when provided)

  In SpacePage.tsx:
    const { user } = useAuth();
    ...
    setGateToken(spaceSlug, data.token);
    if (!user) setToken(data.token);   // only claim global slot if anonymous

  And where SpacePage / EventDetailPage / GalleryPage fetch space events for a
  gated space, pass the gate token override when the logged-in user's own
  request comes back 401/403:
    api.get(`/events?spaceId=${space.id}`, { token: getGateToken(spaceSlug) ?? undefined })

This keeps one concept per key: "photo_token" = who I am, "photo_gate_<slug>"
= what I may view. (A fuller fix is a spaceTokens map in an AuthContext, but
the above is a contained change.)

P2-13 SIGNED-URL COMPARISON IS NOT CONSTANT-TIME
================================================================================
WHERE: functions/lib/signed-url.ts:20 — `return expected === signature;`
(Already tracked as fix-queue #8.) Use crypto.subtle.verify like the Stripe
webhook does:

  export async function verifyMediaSignature(storageKey: string, signature: string, expires: string, secret: string): Promise<boolean> {
    const exp = parseInt(expires, 10);
    if (isNaN(exp) || exp < Math.floor(Date.now() / 1000)) return false;
    const data = `${storageKey}:${exp}`;
    const key = await crypto.subtle.importKey("raw", new TextEncoder().encode(secret), { name: "HMAC", hash: "SHA-256" }, false, ["verify"]);
    // signature is hex; must be even-length hex of the right size
    if (!/^[0-9a-f]{64}$/i.test(signature)) return false;
    const sigBytes = new Uint8Array(signature.match(/.{2}/g)!.map(h => parseInt(h, 16)));
    return crypto.subtle.verify("HMAC", key, sigBytes, new TextEncoder().encode(data));
  }
