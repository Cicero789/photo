NEW-2 (P1) LINT GATE FAILS: 1 error + 57 warnings — pre-existing, but now
  confirmed (v1 misreported this as passing).
  - THE ERROR: src/pages/InspirationMapPage.tsx:268 has
      // eslint-disable-line react-hooks/exhaustive-deps
    but eslint.config.js never registers eslint-plugin-react-hooks (it IS in
    devDependencies) → "Definition for rule 'react-hooks/exhaustive-deps'
    was not found". FIX — register the plugin for src files in
    eslint.config.js:
      import reactHooks from "eslint-plugin-react-hooks";
      // inside the src/**/*.tsx config object:
      plugins: { "@typescript-eslint": tseslint, "react-hooks": reactHooks },
      rules: {
        "@typescript-eslint/no-floating-promises": "warn",
        "react-hooks/rules-of-hooks": "error",
        "react-hooks/exhaustive-deps": "warn",
      },
  - THE 57 WARNINGS: @typescript-eslint/no-floating-promises across src/
    pages/components (fire-and-forget fetch chains). Because package.json
    lint uses --max-warnings 0, ANY warning fails the gate. Mechanical fix:
    prefix intentional fire-and-forget calls with `void `, e.g.
      void fetchUsers();
    or add .catch(() => {}) where a rejection handler is genuinely wanted.
    Do this as one sweep; keep the rule on.
