EXECUTIVE SUMMARY
-----------------
FrameNest is a multi-tenant photo platform that stores private family photos
and facilitates monetary transactions via Stripe Connect. The architecture
demonstrates deliberate security awareness: all SQL is parameterized, passwords
use PBKDF2 with 100K iterations, file uploads are validated by magic bytes,
Stripe webhooks are HMAC-verified, and security headers (CSP, HSTS, nosniff,
X-Frame-Options) are present on every response.

This audit found ONE critical vulnerability (shared JWT/media signing key),
THREE high-severity issues (non-atomic signup, unvalidated portfolio uploads,
incomplete rate limiting), and SIX medium-severity defense-in-depth gaps.

The system should not handle real user data and real money until the critical
and high findings are remediated.

CRITICAL FINDINGS
================================================================================

C-01  JWT_SECRET reused as media signing key
----------------------------------------------------------------------
WHERE:    functions/api/media/[[path]].ts:42
          functions/lib/signed-url.ts:3-21
VERIFIED: Yes (code read)
WHAT:     The same JWT_SECRET is used for both JWT authentication tokens
          AND HMAC-SHA256 signed media URLs. This violates the cryptographic
          principle of key separation. If either signing mechanism is
          compromised, both authentication and media access are broken.
IMPACT:   Single point of failure for all security guarantees.
FIX:      Add a dedicated MEDIA_SIGNING_SECRET environment variable.
          Derive it independently or use HKDF with the JWT_SECRET + a
          purpose-specific context string.


================================================================================
HIGH FINDINGS
================================================================================

H-01  Signup creates user + space + membership non-atomically
----------------------------------------------------------------------
WHERE:    functions/api/auth/signup.ts:72-96
          functions/api/connections/index.ts:76-87
VERIFIED: Yes (code read)
WHAT:     Three sequential INSERT statements without db.batch(). If any
          fails mid-way, orphaned records remain (user without space,
          space without membership). The connections endpoint also auto-
          creates users with passwords that are never communicated.
IMPACT:   Data integrity violation on critical user-facing path.
FIX:      Wrap in db.batch([...]) for atomic execution.

H-02  Rate limiting does not cover all write endpoints
----------------------------------------------------------------------
WHERE:    functions/_middleware.ts:46-51
VERIFIED: Yes (code read + prefix list analysis)
WHAT:     Rate limiting covers 9 prefix patterns but misses:
          POST /api/events, POST /api/connections (sends email!),
          POST /api/bookings/inquire, POST /api/auth/reset-password,
          POST /api/spaces/members, POST /api/admin/seed-inspiration
IMPACT:   Resource exhaustion, email relay abuse via connections.
FIX:      Rate-limit all POST/PUT/DELETE by default; allowlist GETs.

H-03  Portfolio upload has no content validation
----------------------------------------------------------------------
WHERE:    functions/api/photographers/portfolio.ts:36-41
VERIFIED: Yes (code read)
WHAT:     Unlike photos/upload.ts and videos/upload.ts which call
          validateUploadContent(), the portfolio endpoint does NOT
          validate file content, MIME type, or size. An attacker can
          upload HTML/SVG with embedded scripts.
IMPACT:   Stored content injection under the application's origin.
FIX:      Add validateUploadContent() + MIME + size checks.


================================================================================
MEDIUM FINDINGS
================================================================================

M-01  Dev JWT secret used in preview/non-production deployments
----------------------------------------------------------------------
WHERE:    functions/lib/jwt.ts:3,47-54
WHAT:     Hardcoded DEV_SECRET falls back when ENVIRONMENT != "production"
          and JWT_SECRET is unset. Preview deployments on Cloudflare Pages
          are publicly accessible by default.
FIX:      Require JWT_SECRET in all environments; fail closed.

M-02  CSP allows unsafe-inline for styles
----------------------------------------------------------------------
WHERE:    functions/_middleware.ts:29
WHAT:     style-src includes 'unsafe-inline', weakening CSP against CSS
          injection attacks.
FIX:      Use nonce-based or hash-based style-src policy.

M-03  Permissive localhost CORS origin check
----------------------------------------------------------------------
WHERE:    functions/_middleware.ts:19
WHAT:     origin.startsWith("http://localhost") matches
          http://localhost.evil.com as well as legitimate localhost.
FIX:      Parse URL and validate hostname === "localhost".

M-04  Album password transmitted in URL query string
----------------------------------------------------------------------
WHERE:    src/pages/AlbumViewerPage.tsx:33
WHAT:     Password sent as ?pw= query parameter, visible in browser
          history, Referer headers, and server logs.
FIX:      Send via X-Album-Password header instead.

M-05  DeepSeek AI summary is fire-and-forget with silent failure
----------------------------------------------------------------------
WHERE:    functions/api/events/index.ts:74
WHAT:     .catch(() => {}) swallows AI generation errors silently.
FIX:      Log errors; consider retry mechanism.

M-06  Race condition on review duplicate check
----------------------------------------------------------------------
WHERE:    functions/api/photographers/reviews/[slug].ts:37-41
WHAT:     SELECT + INSERT without UNIQUE constraint. Two simultaneous
          requests can create duplicate reviews.
FIX:      Add UNIQUE(photographer_id, reviewer_id) to reviews table.
